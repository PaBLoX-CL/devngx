    What's this all about?

    This is a very simple set of bash scripts that automate a bit my workflow to setup local environments. As you can see, it's quite opinionated, and I basically use it

    Requirements

    • foreman
    • A dns server (I recommend dnsmasq)
    • nginx
    • bash
    • mktemp

    How-to

    Setup

    Besides having First you have to make sure that your dns server catches all calls to *.dev. In the case of dnsmasq, it's enough adding:

    # /etc/dnsmasq.conf
address=/dev/127.0.0.1

    My nginx configuration is something like this:

    # /etc/nginx/nginx.conf
http { 
    …
    include sites/enabled/*.conf;
    …
}

$ tree /etc/nginx/sites
/etc/nginx/sites
├── available
│   ├── pablox.conf
│   └── resp.conf
└── enabled
    ├── pablox.conf -> ../available/pablox.conf
    └── resp.conf -> ../available/resp.conf

    Usage

    $ git clone https://github.com/PaBLoX-CL/devngx && cd devng
$ ./devngx rails-api ~/documentation/rails-api activate # the last argument is optional
‘/tmp/pablo/tmp.lWY5EvdHDe/rails-api.nginx.conf’ -> ‘/etc/nginx/sites/available/rails-api.conf’
‘/etc/nginx/sites/enabled/rails-api.conf’ -> ‘/etc/nginx/sites/available/rails-api.conf’
‘/tmp/pablo/tmp.lWY5EvdHDe/rails-api.Procfile’ -> ‘/home/pablo/Documents/rails-api/Procfile’
‘/tmp/pablo/tmp.lWY5EvdHDe/rails-api.Procfile.dev’ -> ‘/home/pablo/Documents/rails-api/Procfile.dev’
‘/tmp/pablo/tmp.lWY5EvdHDe/rails-api.env’ -> ‘/home/pablo/Documents/rails-api/.env’
$ sudo nginx -s reload
$ rm Procfile.dev # not needed

    Now we need to make a modification (assuming you already have a rails application with the docs built... here it is a pretty good article on how to do it):

    $ cat /etc/nginx/enabled/rails-api.dev
server {
  listen 80;
  server_name rails-api.dev;

  location / {
    proxy_pass http://127.0.0.1:3003;
  }
}
$ cat .env
PORT=3003
$ cat Procfile
server: bundle exec rackup -p $PORT
$ foreman start
22:32:28 server.1 | started with pid 11692
22:32:29 server.1 | [2014-10-08 22:32:29] INFO  WEBrick 1.3.1
22:32:29 server.1 | [2014-10-08 22:32:29] INFO  ruby 2.1.2 (2014-05-08) [x86_64-linux]
22:32:29 server.1 | [2014-10-08 22:32:29] INFO  WEBrick::HTTPServer#start: pid=11692 port=3003

    Go to your browser, write http://rails-api.dev, TADÁ!

    Gotchas

    The method to keep count of which port we are using it's pretty basic:

    $ cat ~/.config/devngx/conf
port=3004

    Stopping the app in the middle, could led to weird issues, specially if it's stopped before reaching the count is augmented (see _fin.sh.
